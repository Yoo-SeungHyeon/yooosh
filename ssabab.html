  • • 실시간 메뉴 정보 제공
  • • 모니터링 기반 서비스 최적화

기술적 도전과 해결

⚡

도전 1: 점심시간 트래픽 집중 문제

상황

점심시간(12:00-13:00)에 전체 교육생들이 동시에 접속하여 서버 응답 시간이 급격히 증가하는 문제가 발생했습니다.

해결 방법
  • • Redis 캐싱으로 데이터베이스 부하 분산
  • • Kubernetes HPA로 Pod 자동 스케일링
  • • CDN 적용으로 정적 리소스 부하 감소
  • • 데이터베이스 커넥션 풀 최적화
🎯

도전 2: 예측 정확도 향상

상황

초기 단순 평균 기반 예측 모델은 정확도가 60% 수준으로, 사용자들이 신뢰하기 어려운 수준이었습니다.

해결 방법
  • • 날씨, 요일, 시기별 가중치 적용
  • • 개인별 선호도 패턴 학습 알고리즘 도입
  • • 실시간 피드백 반영 시스템 구축
  • • A/B 테스트를 통한 모델 성능 비교

향후 발전 계획

단기 계획 (3개월)

  • • 알림 기능 추가: 좋아하는 메뉴 출시 시 push 알림
  • • 소셜 기능 강화: 친구 따라 먹기, 그룹 추천
  • • 데이터 시각화: 개인별 식습관 분석 리포트

장기 계획 (6개월+)

  • • AI 모델 고도화: 딥러닝 기반 추천 시스템
  • • 다른 캠퍼스 확장: SSAFY 전국 캠퍼스 적용
  • • 오픈 소스화: 다른 교육기관에서도 사용 가능
